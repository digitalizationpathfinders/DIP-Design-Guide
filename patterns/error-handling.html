Pattern

Error handling

Provide clear, consistent error feedback and keep users directed toward fixing issues quickly.

Inline field messages

Show errors next to the affected field and keep the message short. Use accessible HTML attributes to connect the message to the input.

Top-level summary

For long forms, add a top-level error summary so users immediately know what needs attention.

<div class="error-summary" role="alert" aria-labelledby="error-summary-heading">
  <h2 id="error-summary-heading">There are problems with your form</h2>
  <ul>
    <li><a href="#email">Enter a valid email address</a></li>
  </ul>
</div>

Portal-specific note

Inside portals, verify whether the portal content region uses a fixed width or a framed panel for error messaging. Outside portals, use Canada.ca guidance for page-level errors.
